# Agent guide — Rosé Pine Glamour

This file orients AI agents working in **rose-pine-glamour**: canonical
[Glamour](https://github.com/charmbracelet/glamour) styles for the [Rosé Pine](https://rosepinetheme.com/) palette.

## Project purpose

- Define Rosé Pine markdown rendering styles for terminal apps (Glow, custom CLIs).
- Keep Go sources **upstream-shaped** so they can be copied into `charmbracelet/glamour` with minimal changes.
- Export `styles/*.json` for consumers that load JSON stylesheets.

Built-in Glow themes live in Glamour, not Glow. Upstream target: **charmbracelet/glamour**.

## See also

This repo is standalone: Glamour styles (Go → JSON), build, test, and preview. Do not add Glow installer logic or
`glow.yml` tooling here.

## Architecture

```text
styles/palette.go          Official palette hex values per variant
styles/builder.go          Shared ansi.StyleConfig builder (all elements + chroma)
styles/rose-pine*.go       One file per variant → StyleConfig var
styles/styles.go           Style name constants + DefaultStyles map
styles/generate.go         //go:generate → JSON export
internal/generate-style-json/   Writes styles/*.json
scripts/verify/            Renders sample markdown via Glamour
examples/sample.md         Preview fixture
Common.make                Shared make targets (included by Makefile)
scripts/log.bash           Rosé Pine–themed Makefile output
scripts/generate-screenshots.py   Gallery PNG generator (optional Python dev tool)
```

**Source of truth:** Go files under `styles/`. JSON is generated output.

## Style variants

| Constant / CLI name   | File                     | Notes                                                                                             |
| --------------------- | ------------------------ | ------------------------------------------------------------------------------------------------- |
| `rose-pine`           | `rose-pine.go`           | Official main (dark)                                                                              |
| `rose-pine-moon`      | `rose-pine-moon.go`      | Official moon base                                                                                |
| `rose-pine-moon-dark` | `rose-pine-moon-dark.go` | Moon syntax on main base; pairs with rose-pine-bat — community variant, weaker upstream candidate |
| `rose-pine-dawn`      | `rose-pine-dawn.go`      | Official light                                                                                    |

For upstream Glamour PRs, prefer starting with **`rose-pine`** and **`rose-pine-dawn`** unless maintainers agree to
more.

## Commands

Targets live in `Common.make` (included from `Makefile`). Output uses Rosé Pine–themed `scripts/log.bash`.

```bash
make help        # list targets (Rosé Pine–themed output)
make check       # verify jq and go (glow/python optional)
make build       # go generate → styles/*.json
make test        # validate JSON + Glamour render (scripts/verify)
make screenshots # gallery PNGs (Python + Pillow — dev tooling only)
make preview     # glow examples/sample.md (STYLE=rose-pine-dawn)
make clean       # Python caches and Go test cache
```

Always run **`make test`** after palette or builder changes.

## Editing guidelines

### Changing colors

1. Edit hex values in `styles/palette.go` only — stay aligned with
   [rose-pine/palette](https://github.com/rose-pine/palette).
2. If syntax role mapping changes, edit `styles/builder.go` (shared across variants).
3. Run `make build` then `make test`.
4. If rose-pine-bat uses the same role, consider updating rose-pine-bat for parity.

### Adding a variant

1. Add palette in `palette.go`.
2. Add `styles/rose-pine-<name>.go` with `var …StyleConfig = buildStyleConfig(…Palette)`.
3. Register constant + map entry in `styles/styles.go`.
4. Run `make build` and `make test`.
5. Update README variant table and `examples/sample.md` if relevant.

### Do not

- Hand-edit `styles/*.json` — they are regenerated by `go generate`.
- Add Glow installer logic or `glow.yml` tooling here (see
  [rose-pine-glow](https://github.com/drluckyspin/rose-pine-glow)).
- Use the Rose Pine template’s `template.json` / `@rose-pine/build` — Glamour needs Go `ansi.StyleConfig`.
- Treat Python as part of the core style pipeline — it is only for `make screenshots` gallery assets.
- Introduce unrelated refactors when fixing palette or style issues.

### Upstream Glamour shape

Match existing themes (e.g. `dracula.go` in glamour):

- Import `charm.land/glamour/v2/ansi`
- One `*StyleConfig` var per theme file
- Register in `DefaultStyles` with hyphenated names (`rose-pine`, not `rose_pine`)
- Run glamour’s `go generate ./...` after adding gallery PNGs under `styles/gallery/`

## Dependencies

- **Root `go.mod`:** `charm.land/glamour/v2` — types for style definitions and JSON export.
- **`scripts/verify/go.mod`:** `github.com/charmbracelet/glamour` — runtime render check (may differ from v2 import
  path; verify script only).

## CI

`.github/workflows/validate.yml` runs `make test` on push/PR.

## Upstreaming checklist

1. Open issue on charmbracelet/glamour (scope: which variants).
2. Copy `styles/*.go` (+ shared `palette.go` / `builder.go` or inline per upstream preference).
3. Register in glamour’s `styles/styles.go`.
4. `go generate ./...`, add gallery screenshots, run glamour tests (`go test ./pkg/...`).
5. List on [rose-pine-site](https://github.com/rose-pine/rose-pine-site) when released.

## User-facing docs

- **README.md** — install, usage, variant table; keep in sync with code changes.
- **AGENTS.md** (this file) — agent orientation; update when workflow or layout changes.

## Common pitfalls

- **Glow TUI + custom JSON:** use absolute paths in `glow.yml`; tilde expansion is unreliable in TUI.
- **Env vars:** TUI uses `GLAMOUR_STYLE`, not `GLOW_STYLE`.
- **JSON drift:** if JSON is committed, always regenerate after Go edits so CI and consumers stay consistent.
